# 第二阶段：入库对账基础功能方案

## 1. 阶段目标

第二阶段目标是在已完成登录、账号、主界面框架的基础上，先做出“入库对账”的基础可用版本。

本阶段重点是打通基础链路：

```text
选择药师帮对账单 Excel
  ↓
读取本月支付账单明细
  ↓
配置 MySQL 和入库明细 SQL
  ↓
查询入库明细
  ↓
执行基础商品级对账
  ↓
导出对账结果 Excel
```

第三阶段再扩展为两个独立按钮：

- 供应商对账；
- 供应商商品对账。

第三阶段详细规则见：

[第三阶段_供应商与供应商商品对账方案.md](./第三阶段_供应商与供应商商品对账方案.md)

## 2. 本阶段范围

### 2.1 包含

- 药师帮对账单 Excel 读取；
- 识别 `本月支付账单明细`；
- 药师帮商品明细标准化；
- MySQL 数据库配置；
- 数据库连接测试；
- 入库明细 SQL 配置；
- SQL 只读安全校验；
- SQL 预览；
- 入库明细标准化；
- 基础商品级匹配；
- 数量、单价、金额核对；
- 差异结果导出；
- 对账任务记录。

### 2.2 暂不包含

- 供应商对账；
- 供应商商品对账；
- 将入库商品编码回填到药师帮明细；
- 供应商 + 商品编码汇总金额对账；
- 商品映射维护；
- 自动下载药师帮对账单；
- 自动修改入库系统数据。

## 3. 用户操作流程

```text
进入入库对账页面
  ↓
选择药师帮对账单 Excel
  ↓
系统识别本月支付账单明细
  ↓
选择 MySQL 数据库配置
  ↓
编辑或选择入库明细 SQL
  ↓
测试连接并预览 SQL
  ↓
选择账期和入库查询日期范围
  ↓
执行基础对账
  ↓
查看结果概览
  ↓
导出对账结果 Excel
```

## 5. 药师帮数据读取

### 5.1 工作表

本阶段主数据：

- `本月支付账单明细`

其他工作表先作为预留：

- `本月支付订单`
- `往月订单本月退款`

### 5.2 关键字段

需要读取：

- 订单号；
- 订单类型；
- 药店名称；
- 供应商；
- 企业名称；
- 采购时间；
- 商品名称；
- 厂家；
- 规格；
- 单位；
- 批准文号；
- 条形码；
- 批号；
- 生产日期；
- 有效期；
- 单价；
- 折后价；
- 下单数量；
- 退款数量；
- 折后价总额；
- 商品总金额；
- 运费；
- 优惠金额。

### 5.3 标准字段

- ysb_order_no：药师帮订单号；
- ysb_supplier_name：药师帮供应商；
- purchase_time：采购时间；
- product_name：商品名称；
- manufacturer：厂家；
- spec：规格；
- barcode：条形码；
- batch_no：批号；
- expiry_date：有效期；
- unit_price：折后价；
- quantity：净采购数量；
- amount：折后价总额；
- raw_quantity：下单数量；
- refund_quantity：退款数量。

### 5.4 清洗规则

- 商品名称、规格、厂家、供应商去除前后空格；
- 条形码统一转为文本；
- 数量为空按 0 处理；
- 金额为空按 0 处理；
- 日期统一转为标准日期；
- 过滤空商品行；
- 保留原始行号。

## 6. 入库系统数据读取

### 6.1 数据库配置

配置项：

- 配置名称；
- 主机地址；
- 端口；
- 数据库名；
- 用户名；
- 密码；
- 字符集；
- 连接超时时间；
- 是否启用；
- 入库明细 SQL。

### 6.2 SQL 安全规则

- SQL 必须以 SELECT 开头；
- 禁止 INSERT、UPDATE、DELETE、DROP、TRUNCATE、ALTER；
- 禁止多语句执行；
- 推荐使用只读数据库账号。

### 6.3 SQL 参数

入库明细 SQL 建议支持：

- start_date；
- end_date。

### 6.4 入库标准字段

入库 SQL 必须至少输出：

- inbound_no：入库单号；
- inbound_date：入库日期；
- product_name：商品名称；
- manufacturer：生产厂家；
- spec：规格；
- quantity：入库数量；
- unit_price：入库单价；
- amount：入库金额。

建议输出：

- supplier_name；
- product_code；
- barcode；
- batch_no；
- expiry_date；
- document_status。

## 7. 基础对账规则

### 7.1 匹配优先级

第一版基础匹配优先级：

1. 条形码一致；
2. 商品编码或药师帮编码一致；
3. 商品名称 + 规格 + 厂家一致；
4. 无法唯一匹配时标记为“需要人工确认”。

### 7.2 聚合规则

同一商品可能存在多行采购和多行入库。

基础版本先按商品匹配键聚合：

- 数量求和；
- 金额求和；
- 单价可使用金额 / 数量计算；
- 订单号合并展示；
- 入库单号合并展示。

### 7.3 差异判断

差异字段：

- 数量；
- 单价；
- 金额。

差异类型：

- 一致；
- 入库系统缺少；
- 药师帮缺少；
- 数量不一致；
- 单价不一致；
- 金额不一致；
- 需要人工确认。

误差规则：

- 单价误差以界面配置为准；
- 金额误差以界面配置为准；
- 是否检查数量以界面配置为准。

- 差异类型；
- 匹配方式；
- 药师帮订单号；
- 入库单号；
- 药师帮供应商；
- 入库供应商；
- 商品名称；
- 规格；
- 厂家；
- 条形码；
- 药师帮数量；
- 入库数量；

- 药师帮单价；

## 11. 验收标准

第二阶段完成后，需要满足：

- 能选择药师帮对账单 Excel；
- 能识别 `本月支付账单明细`；
- 能读取药师帮商品明细；
- 条形码不会变成科学计数法；
- 能配置 MySQL 数据库；
- 能测试数据库连接；
- 能保存入库明细 SQL；
- SQL 只能执行 SELECT；
- SQL 预览能显示前 20 行；
- 缺少必要字段时能提示；
- 能按日期范围查询入库明细；
- 能执行基础商品级匹配；
- 能核对数量、单价、金额；
- 能生成对账结果 Excel；
- 结果 Excel 包含 `汇总`、`对账明细`、`差异明细`；
- 差异行有明显高亮；
- 对账任务能在执行记录中查看；
- 不对入库系统数据库执行任何写操作。

## 12. 待确认问题

1. 入库系统 MySQL 表结构；
2. 是否已有只读数据库账号；
3. 入库系统是否保存条形码；
4. 入库系统是否保存药师帮编码；
5. 药师帮退款数量是否要从下单数量中扣除；
6. 往月订单本月退款是否参与本月对账；
7. 入库查询日期需要比采购账期放宽几天；
8. 单价误差界面初始值设置为多少；
9. 金额误差界面初始值设置为多少；
10. 数量差异是否作为严重差异，还是仅提示。
