# 第三阶段：供应商与供应商商品对账方案

## 1. 阶段目标

第三阶段在第二阶段基础入库对账能力完成后，新增两个独立对账功能：

1. 供应商对账；
2. 供应商商品对账。

这两个功能在界面上应做成两个独立按钮：

- `供应商对账`
- `供应商商品对账`

二者不是“两层流程”，也不是必须前后执行。用户可以根据业务需要分别点击、分别导出、分别查看任务记录。

## 2. 功能一：供应商对账

### 2.1 数据来源

药师帮侧：

- 使用 `本月支付订单`。

入库系统侧：

- 使用入库明细 SQL 查询结果。

### 2.2 对账规则

供应商对账规则：

```text
入库明细按供应商汇总金额
vs
药师帮 本月支付订单 按供应商汇总金额
```

药师帮侧聚合维度：

- 药师帮供应商。

入库侧聚合维度：

- 入库供应商名称。

聚合字段：

- 药师帮供应商金额合计；
- 入库供应商金额合计；
- 金额差异 = 入库供应商金额合计 - 药师帮供应商金额合计；
- 药师帮订单数；
- 入库明细行数。

供应商金额误差以界面配置为准，界面初始值为 1 元。

### 2.3 药师帮金额口径

供应商对账中，药师帮 `本月支付订单` 使用金额字段：

- 实际支付金额，且该金额按业务口径视为已减退款。

入库侧使用金额字段：

- `amount`。

### 2.4 差异类型

- 供应商金额一致；
- 供应商金额不一致；
- 入库系统缺少供应商；
- 药师帮缺少供应商；
- 供应商名称疑似不一致；
- 需要人工确认。

### 2.5 导出结果

供应商对账导出文件名：

```text
药师帮供应商对账结果_账期_生成时间.xlsx
```

至少包含工作表：

- `汇总`
- `供应商对账`
- `差异明细`

供应商对账页字段：

- 对账状态；
- 药师帮供应商；
- 入库供应商；
- 药师帮金额；
- 入库金额；
- 金额差异；
- 药师帮订单数；
- 入库明细行数；
- 匹配方式；
- 备注。

## 3. 功能二：供应商商品对账

### 3.1 数据来源

药师帮侧：

- 使用 `本月支付账单明细`。

入库系统侧：

- 使用入库明细 SQL 查询结果。

### 3.2 对账规则

供应商商品对账规则：

```text
先通过条形码、商品名称、规格、厂家进行模糊匹配
↓
将入库供应商和入库商品编码回填到药师帮明细
↓
药师帮侧按 供应商 + 商品编码 汇总金额
↓
入库侧按 供应商 + 商品编码 汇总金额
↓
核对金额和数量
```

最终对账维度：

- 供应商；
- 商品编码。

药师帮明细金额口径：

- 使用 `折后价总额`。

药师帮明细数量口径：

- 净采购数量 = 下单数量 - 退款数量。

退款处理：

- 退款数量从下单数量中扣除；
- `往月订单本月退款` 参与本月对账。

### 3.3 明细模糊匹配

匹配输入字段：

- 条形码；
- 商品名称；
- 规格；
- 厂家；
- 供应商。

匹配优先级：

1. 条形码完全一致；
2. 条形码为空或未命中时，按商品名称 + 规格 + 厂家模糊匹配；
3. 如果启用供应商约束，则只在同供应商或疑似同供应商范围内匹配；
4. 多个候选同时命中时，选择分数最高且唯一领先的候选；
5. 分数不足或候选不唯一时，标记为“需要人工确认”。

匹配阈值以界面配置为准：

- 自动通过阈值，界面初始值暂定为 85；
- 疑似匹配阈值，界面初始值暂定为 70。

### 3.4 编码回填

匹配结果需要写入药师帮明细结果：

- 匹配到的入库供应商；
- 匹配到的入库商品编码；
- 匹配到的入库商品名称；
- 匹配到的入库规格；
- 匹配到的入库厂家；
- 匹配方式；
- 匹配分数；
- 匹配状态；
- 匹配备注。

第一版不直接覆盖原始药师帮 Excel，而是导出带回填字段的新结果文件。

### 3.5 供应商商品汇总

药师帮侧聚合维度：

- matched_supplier_name；
- matched_inbound_product_code。

入库侧聚合维度：

- supplier_name；
- product_code。

聚合字段：

- 药师帮金额合计；
- 入库金额合计；
- 金额差异 = 入库金额合计 - 药师帮金额合计；
- 药师帮数量合计；
- 入库数量合计；
- 数量差异 = 入库数量合计 - 药师帮数量合计。

误差规则：

- 供应商商品金额误差以界面配置为准，界面初始值为 1 元；
- 数量误差以界面配置为准，界面初始值为 1 个；
- 数量差异仅提示，不作为严重差异。

### 3.6 差异类型

- 供应商商品金额一致；
- 供应商商品金额不一致；
- 供应商商品数量不一致；
- 药师帮明细未匹配商品编码；
- 入库系统存在但药师帮缺少；
- 药师帮存在但入库系统缺少；
- 需要人工确认。

### 3.7 导出结果

供应商商品对账导出文件名：

```text
药师帮供应商商品对账结果_账期_生成时间.xlsx
```

至少包含工作表：

- `汇总`
- `药师帮明细编码回填`
- `供应商商品对账`
- `差异明细`

药师帮明细编码回填页字段：

- 原始行号；
- 订单号；
- 药师帮供应商；
- 商品名称；
- 规格；
- 厂家；
- 条形码；
- 药师帮数量；
- 药师帮金额；
- 匹配到的入库供应商；
- 匹配到的入库商品编码；
- 匹配到的入库商品名称；
- 匹配到的入库规格；
- 匹配到的入库厂家；
- 匹配方式；
- 匹配分数；
- 匹配状态；
- 匹配备注。

供应商商品对账页字段：

- 对账状态；
- 供应商；
- 商品编码；
- 商品名称；
- 规格；
- 厂家；
- 药师帮金额；
- 入库金额；
- 金额差异；
- 药师帮数量；
- 入库数量；
- 数量差异；
- 备注。

## 4. 界面调整

第三阶段在入库对账页面增加两个独立按钮：

- `供应商对账`
- `供应商商品对账`

供应商对账配置：

- 供应商金额误差；
- 供应商名称标准化开关；
- 供应商名称模糊匹配开关；
- 药师帮订单金额口径。

供应商商品对账配置：

- 条形码匹配开关；
- 商品名称匹配开关；
- 规格匹配开关；
- 厂家匹配开关；
- 供应商限制开关；
- 模糊匹配自动通过阈值，默认 85；
- 疑似匹配阈值，默认 70；
- 供应商商品金额误差；
- 数量误差；
- 是否检查数量。

界面初始值：

- 供应商金额误差：1 元；
- 供应商商品金额误差：1 元；
- 数量误差：1 个；
- 数量差异：仅提示；
- 入库查询日期放宽天数：根据界面配置；
- 供应商约束明细候选范围：优先启用；
- 模糊匹配自动通过阈值：85；
- 疑似匹配阈值：70；
- 商品映射表维护：不需要。

所有误差和阈值都以界面配置为准，对账引擎不得写死。

### 4.1 模糊匹配阈值说明

模糊匹配有两个阈值，需要界面配置：

- 自动通过阈值：匹配分数达到或超过该值时，系统自动认为商品匹配成功，并回填入库商品编码。
- 疑似匹配阈值：匹配分数低于自动通过阈值、但达到或超过该值时，系统不自动确认，标记为“疑似匹配 / 需要人工确认”。

低于疑似匹配阈值时，标记为“未匹配”。

当前暂定界面初始值：

```text
自动通过阈值 = 85
疑似匹配阈值 = 70

分数 >= 85：自动匹配
70 <= 分数 < 85：疑似匹配，需要人工确认
分数 < 70：未匹配
```

上述 85 和 70 为暂定业务初始值，后续如实际匹配结果偏严或偏松，可通过界面配置调整。

## 7. 验收标准

第三阶段完成后，需要满足：

- 界面存在 `供应商对账` 和 `供应商商品对账` 两个按钮；
- 两个按钮可以独立执行；
- 供应商对账使用 `本月支付订单`；
- 供应商商品对账使用 `本月支付账单明细`；
- 点击 `供应商对账` 后，能用入库明细按供应商汇总，与 `本月支付订单` 按供应商汇总后的金额核对；
- 供应商对账能导出 `汇总`、`供应商对账`、`差异明细`；
- 点击 `供应商商品对账` 后，能通过条形码、商品名称、规格、厂家进行明细模糊匹配；
- 能将入库供应商和入库商品编码回填到药师帮明细结果中；
- 能标记自动匹配、疑似匹配、未匹配；
- 能按供应商 + 商品编码汇总金额执行核对；
- 供应商商品对账能导出 `汇总`、`药师帮明细编码回填`、`供应商商品对账`、`差异明细`；
- 两类对账任务都能在执行记录中查看；
- 不对入库系统数据库执行任何写操作。

## 8. 已确认业务口径

1. 供应商对账中，药师帮 `本月支付订单` 使用 `实际支付金额`，且该金额按业务口径视为已减退款；
2. 供应商对账中，入库侧使用 `amount`；
3. 供应商商品对账中，药师帮明细使用 `折后价总额`；
4. 退款数量从下单数量中扣除；
5. `往月订单本月退款` 参与本月对账；
6. 入库查询日期比采购账期放宽几天，根据界面配置走；
7. 供应商金额误差界面初始值为 1 元；
8. 供应商商品金额误差界面初始值为 1 元；
9. 数量误差界面初始值为 1 个；
10. 数量差异仅提示，不作为严重差异；
11. 不需要后续维护商品映射表；
12. 明细候选范围优先使用供应商约束；
13. 模糊匹配自动通过阈值界面初始值暂定为 85；
14. 疑似匹配阈值界面初始值暂定为 70。
