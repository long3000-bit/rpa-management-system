# 第一阶段：登录界面、用户、密码等基础功能方案

## 1. 阶段目标

第一阶段先完成软件的基础入口能力，目标是让软件具备一个可运行、可登录、可管理基础账号的本地桌面系统骨架。

本阶段完成后，应具备：

- 软件启动页；
- 登录界面；
- 本地用户账号；
- 密码校验；
- 修改密码；
- 退出登录；
- 记住账号；
- 基础权限字段预留；
- 登录日志；
- 主界面框架；
- 后续 RPA 机器人、入库对账模块的菜单入口。

第一阶段不急着接完整 RPA 和对账业务，先把“谁能进系统、进来后看到什么、用户信息怎么保存”做好。

## 2. 功能范围

### 2.1 包含功能

第一版登录与账号功能包含：

- 登录页面；
- 用户名和密码输入；
- 密码显示 / 隐藏切换；
- 记住用户名；
- 登录失败提示；
- 默认管理员账号初始化；
- 修改密码；
- 退出登录；
- 登录成功进入主界面；
- 主界面左侧菜单占位；
- 当前登录用户显示；
- 登录日志记录。

### 2.2 暂不包含功能

第一阶段暂不做：

- 多角色复杂权限；
- 手机短信登录；
- 验证码；
- 人脸识别；
- 远程服务器账号体系；
- 企业微信 / 钉钉登录；
- 密码找回；
- 多人同时在线控制；
- 云端同步。

这些能力后续如果需要，可以在账号模块基础上扩展。

## 3. 登录界面设计

### 3.1 页面元素

登录界面建议包含：

- 软件名称；
- 用户名输入框；
- 密码输入框；
- 密码显示 / 隐藏按钮；
- 记住用户名复选框；
- 登录按钮；
- 错误提示区域；
- 版本号；
- 本机数据目录提示入口。

### 3.2 交互规则

- 用户名为空时，不允许登录；
- 密码为空时，不允许登录；
- 用户名或密码错误时，显示明确提示；
- 连续失败时记录失败日志；
- 登录成功后进入主界面；
- 如果勾选“记住用户名”，下次打开软件自动填入用户名；
- 不保存明文密码；
- 回车键触发登录。

### 3.3 默认账号

首次启动软件时，如果本地没有任何用户，则自动初始化默认管理员账号：

```text
用户名：admin
初始密码：admin123
```

首次登录后建议提示修改密码。

正式交付时可把默认账号和初始密码写入部署说明，或在首次启动时由用户自行创建管理员账号。

## 4. 用户管理设计

### 4.1 用户字段

用户表建议字段：

- id；
- username：用户名；
- password_hash：密码哈希；
- salt：密码盐；
- display_name：显示名称；
- role：角色；
- status：状态；
- last_login_at：最后登录时间；
- created_at：创建时间；
- updated_at：更新时间。

### 4.2 用户状态

用户状态建议：

- active：启用；
- disabled：禁用。

第一阶段可以只内置管理员账号，先不开放完整用户管理页面，但数据结构要预留。

### 4.3 角色预留

第一阶段预留角色字段：

- admin：管理员；
- operator：操作员；
- viewer：只读查看。

第一阶段可以先只启用 admin，后续再做角色权限控制。

## 5. 密码安全设计

### 5.1 存储规则

密码不能明文保存。

建议使用：

- PBKDF2；
- bcrypt；
- argon2。

如果第一版希望减少依赖，可先使用 Python 标准库：

```text
hashlib.pbkdf2_hmac
```

保存内容：

- password_hash；
- salt；
- hash_iterations。

### 5.2 密码规则

第一阶段密码规则建议：

- 长度至少 6 位；
- 不允许为空；
- 新密码和确认密码必须一致；
- 修改密码时必须输入旧密码。

后续可以增强为：

- 至少 8 位；
- 包含数字和字母；
- 定期更换；
- 登录失败锁定。

## 6. 修改密码功能

### 6.1 入口

主界面右上角用户菜单提供：

- 修改密码；
- 退出登录。

### 6.2 修改密码表单

字段：

- 当前密码；
- 新密码；
- 确认新密码。

校验：

- 当前密码必须正确；
- 新密码不能为空；
- 新密码不能少于 6 位；
- 两次新密码必须一致；
- 修改成功后提示重新登录或继续使用。

## 7. 主界面基础框架

登录成功后进入主界面。

主界面包含：

- 左侧导航；
- 顶部栏；
- 当前登录用户；
- 内容区域；
- 退出登录入口。

左侧导航第一阶段先做入口占位：

- 首页；
- RPA 机器人；
- 入库对账；
- 执行记录；
- 配置中心；
- 日志与截图。

第一阶段可以只做页面占位，不实现完整业务逻辑。

## 8. 本地数据存储

### 8.1 推荐存储方式

使用 SQLite 保存本地账号和基础配置。

建议数据库文件：

```text
data/app.db
```

### 8.2 数据表

第一阶段至少需要：

```text
users
login_logs
app_settings
```

### 8.3 users 表

```text
id
username
password_hash
salt
hash_iterations
display_name
role
status
last_login_at
created_at
updated_at
```

### 8.4 login_logs 表

```text
id
username
success
message
login_at
machine_name
```

### 8.5 app_settings 表

```text
key
value
updated_at
```

用于保存：

- remembered_username；
- last_login_user；
- first_run_completed；
- app_version。

## 9. 登录流程

```text
启动软件
  ↓
初始化本地目录
  ↓
初始化 SQLite 数据库
  ↓
检查是否存在用户
  ↓
不存在则创建默认管理员
  ↓
显示登录界面
  ↓
输入用户名和密码
  ↓
校验用户是否存在、是否启用、密码是否正确
  ↓
登录成功
  ↓
记录登录日志
  ↓
进入主界面
```

## 10. 异常处理

需要处理以下异常：

- 数据库文件不存在；
- 数据库初始化失败；
- 用户表缺失；
- 默认管理员创建失败；
- 用户名不存在；
- 密码错误；
- 用户被禁用；
- 配置文件损坏。

错误提示要面向普通用户，避免直接暴露技术异常。

例如：

- “用户名或密码错误”
- “当前账号已被禁用”
- “本地数据库初始化失败，请联系管理员”

详细错误写入日志文件。

## 11. 技术实现建议

第一阶段推荐技术：

- 桌面界面：PySide6；
- 本地数据库：SQLite；
- 密码哈希：hashlib.pbkdf2_hmac；
- 日志：logging；
- 配置：SQLite app_settings 表；
- 打包：PyInstaller。

建议代码结构：

```text
RPA/
  app/
    main.py
    ui/
      login_window.py
      main_window.py
      change_password_dialog.py
      pages/
        home_page.py
        rpa_robot_page.py
        reconciliation_page.py
        task_record_page.py
        settings_page.py
        log_page.py
    core/
      auth_service.py
      password_service.py
      settings_service.py
      log_service.py
    storage/
      database.py
      migrations.py
      models.py
  data/
    app.db
  logs/
    app.log
  docs/
```

## 12. 第一阶段开发任务拆分

### 任务 1：项目骨架

- 创建 Python 项目结构；
- 创建 app 目录；
- 创建 main.py；
- 创建基础配置；
- 创建 logs、data 目录。

### 任务 2：数据库初始化

- 创建 SQLite 数据库；
- 创建 users、login_logs、app_settings 表；
- 首次启动自动初始化默认管理员；
- 支持数据库重复初始化不报错。

### 任务 3：密码服务

- 生成 salt；
- 生成密码哈希；
- 校验密码；
- 修改密码。

### 任务 4：认证服务

- 登录校验；
- 用户状态判断；
- 登录成功更新时间；
- 登录日志写入；
- 记住用户名。

### 任务 5：登录界面

- 用户名输入；
- 密码输入；
- 密码显示 / 隐藏；
- 记住用户名；
- 登录按钮；
- 回车登录；
- 错误提示。

### 任务 6：主界面框架

- 登录成功打开主界面；
- 左侧导航；
- 顶部显示当前用户；
- 页面切换；
- RPA 机器人页面占位；
- 入库对账页面占位；
- 配置中心页面占位。

### 任务 7：修改密码与退出登录

- 修改密码弹窗；
- 旧密码校验；
- 新密码确认；
- 退出登录返回登录页。

### 任务 8：打包准备

- 确认本地数据目录；
- 确认日志目录；
- 准备 PyInstaller 打包配置；
- 写一份启动和默认账号说明。

## 13. 第一阶段验收标准

完成后需要满足：

- 双击启动软件后显示登录界面；
- 首次启动会自动生成默认管理员；
- 使用默认账号可以登录；
- 密码错误时不能登录；
- 可以记住用户名；
- 登录成功后进入主界面；
- 主界面能看到 RPA 机器人、入库对账、执行记录、配置中心等菜单；
- 可以修改密码；
- 修改密码后新密码生效；
- 可以退出登录；
- 登录成功和失败都有日志；
- 密码没有明文保存在数据库中。
