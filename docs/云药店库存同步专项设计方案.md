# 云药店库存同步专项设计方案

## 1. 目标定位

云药店库存同步模块是本软件的一个独立功能模块，主要用于对比云药店库存与君元系统库存，并将差异库存通过API接口同步到云药店平台。

业务数据从Excel导入云药店库存数据（需从云药店客户端手动导出），同时从君元系统数据库查询库存数据，按药品编码+批号进行匹配对比，计算出库存差异后调用云药店API接口进行同步，并将同步结果记录到系统数据库，最后按需导出结果Excel。

当前已确认业务前提：

- 目标系统：江阴云药店平台；
- 同步方式：API接口调用（非RPA操作）；
- 登录验证：API签名验证（appkey + appsecret）；
- 主要操作：同步库存；
- 数据来源：Excel导入云药店库存 + 君元系统数据库查询；
- 结果回写：优先写回系统数据库，再按需导出Excel；
- 后续和对账功能共用任务记录、日志、配置中心等基础能力。

**重要说明**：云药店/医保库存查询接口（checkstock）当前服务端关闭，客户端暂时不能通过该接口查询云端库存。因此云药店库存数据必须通过云药店客户端手动导出Excel获取。

本模块第一版目标是实现云药店库存差异自动同步闭环。

## 2. 第一版功能范围

第一版建议完成"Excel导入云药店库存，对比君元库存，API同步差异"的闭环：

1. 选择云药店库存Excel文件；
2. 选择工作表；
3. 预览Excel数据；
4. 配置字段映射（药品编码、批号、库存数量等）；
5. 校验必填字段；
6. 将云药店库存数据导入系统数据库；
7. 配置君元系统数据库连接；
8. 查询君元系统库存数据；
9. 按药品编码+批号匹配对比；
10. 计算库存差异；
11. 配置云药店API参数；
12. 测试API连接；
13. 批量调用API同步差异库存；
14. 记录每条同步结果；
15. 按需导出同步结果Excel；
16. 支持失败重试和断点续跑。

第一版暂不做：

- 自动从云药店平台导出库存（需手动导出Excel）；
- 通过API查询云药店实时库存（接口服务端关闭）；
- 定时自动同步任务；
- 多药店同时同步；
- 库存预警和自动补货；
- 采购入库自动触发同步；
- 销售出库自动触发同步。

## 3. 总体流程

核心流程：

```text
云药店库存Excel（手动导出）
↓
导入模板与字段映射
↓
数据校验
↓
写入本软件数据库
↓
君元系统数据库连接
↓
查询君元库存数据
↓
按药品编码+批号匹配
↓
计算库存差异
↓
筛选需同步的差异记录
↓
调用云药店API同步
↓
记录同步结果
↓
按需导出Excel结果
```

这种设计的好处：

- Excel只作为导入来源，不承担任务状态主库；
- 数据导入后可以在系统中查询、筛选、审核；
- 同步失败后可以直接从系统重试，不依赖人工改Excel；
- 可以保留原始导入数据、对比结果和同步记录，便于追溯；
- 成功后可以直接更新系统业务数据，供后续查询、统计继续使用；
- 不依赖RPA操作，稳定性更高。

**注意事项**：

- 云药店库存Excel必须从云药店客户端手动导出，确保数据时效性；
- 建议在导出Excel后尽快完成同步，避免库存数据过期；
- 同步前请确认Excel库存数据与云药店实际库存一致。

### 4.1 云药店库存导入区

用于把云药店库存Excel导入本软件系统：

- Excel文件选择；
- Sheet选择；
- 字段映射配置；
- 数据预览；
- 校验结果；
- 导入按钮；
- 导入批次号；
- 导入结果统计。

导入原则：

- Excel导入后必须按导入批次保存到底表；
- `库存明细20260223200426.xlsx` 中每个字段都要保留，不能只保存参与对比的字段；
- 底表既保存原始字段，也保存系统标准字段，方便后续查询、对比、导出和追溯；
- 原始整行数据需保存到 `raw_data`，防止后续新增字段时历史导入数据无法追溯；
- 同一导入批次内按Excel原始行号保存，确保能回看原Excel行。

请求头签名规则：

```text

```

请求头字段：

```http

appkey: neusoft
nonce: 随机字符串
timestamp: 当前时间戳
sign: 计算后的MD5签名
```

## 7. 库存对比逻辑

### 7.1 匹配规则

主键匹配：

```text
匹配键 = oldproductno（药品编码） + lotno（批号）
```

匹配优先级：

1. 药品编码 + 批号精确匹配；
2. 药品编码匹配（无批号）；
3. 无法匹配标记为独有。

### 7.2 差异计算

差异计算公式：

```text
diff_quantity = 君元库存数量 - 云药店库存数量
```

差异类型：

| diff_quantity | 说明 | 同步操作 |
|---------------|------|---------|
| > 0 | 君元库存多 | 云药店增加库存 |
| < 0 | 云药店库存多 | 云药店减少库存 |
| = 0 | 库存一致 | 无需同步 |

### 7.3 同步策略

| 场景 | 处理方式 |
|------|---------|
| 云药店库存 < 君元库存 | 调用API增加库存 |
| 云药店库存 > 君元库存 | 调用API减少库存 |
| 云药店库存 = 君元库存 | 跳过，标记一致 |
| 云药店独有 | 可选同步或跳过 |
| 君元独有 | 可选同步或跳过 |

同步时使用synctype=2（差异同步），productqty传差额。

### 7.4 数据时效性说明

由于查询接口暂不可用，云药店库存数据完全依赖Excel导入：

- **建议**：导出Excel后尽快完成同步，避免库存数据过期；
- **风险**：如果Excel导出时间与同步时间间隔较长，云药店实际库存可能已发生变化；
- **对策**：同步前请人工确认Excel库存数据与云药店实际库存一致；
- **后续**：待查询接口恢复后，可增加实时库存校验功能。

## 10. 异常处理

常见异常和处理方式：

| 异常 | 处理 |
|------|------|
| Excel文件打不开 | 提示关闭Excel后重试，数据不入库 |
| 必填字段为空 | 当前行失败，写回原因 |
| 药品编码不存在 | 标记为独有，可选跳过或同步 |
| 批号不存在 | 当条失败，写回原因 |
| API连接失败 | 暂停任务，提示检查网络 |
| API签名错误 | 停止任务，提示检查appsecret |
| API返回错误 | 当条失败，写回错误消息 |
| 云药店库存不足（减少时） | 当条失败，写回原因 |
| 网络超时 | 重试，仍失败则当条失败 |
| 数据库连接失败 | 暂停任务，提示检查连接 |

失败处理策略：

- 单条数据问题：当条失败，继续下一条；
- API连接失败：暂停整个任务；
- 签名错误：停止整个任务；
- 批号不存在：当条失败，继续下一条；
- 云药店库存不足：当条失败，继续下一条。

## 16. 方案优势

1. **不依赖RPA操作**：直接调用API，稳定性更高；
2. **可追溯**：所有同步记录保存到数据库；
3. **可重试**：失败记录可单独重试；
4. **可审计**：对比结果和同步日志完整记录；
5. **可扩展**：支持其他云药店API对接；
6. **可配置**：API参数、匹配规则可灵活配置；
7. **可监控**：实时日志和进度显示。

## 17. 方案限制

由于查询接口暂不可用，当前方案存在以下限制：

1. **数据时效性**：云药店库存数据依赖手动导出Excel，可能存在时效性问题；
2. **人工确认**：同步前需人工确认Excel库存数据与云药店实际库存一致；
3. **无法实时校验**：无法通过API实时查询云药店库存进行校验；
4. **风险提示**：如果Excel导出时间与同步时间间隔较长，可能导致库存同步错误。

**建议**：

- 导出Excel后尽快完成同步；
- 同步前人工核对关键药品库存；
- 待查询接口恢复后，增加实时库存校验功能。
