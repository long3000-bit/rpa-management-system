# 江阴云药店库存同步接口调用文档

适用范围：GSP 客户端“江阴云药店对接”插件中的库存同步功能。

插件文件：

```text
plugins\GSP.Plugin.JYSI.dll
```

配置文件：

```text
plugins\JYSIConfig.ini
```

当前配置：

```text
host = http://61.177.139.195:21456/
appkey = neusoft
appsecret = f7377865580a02f9a89533b31e4ca7b7
orgcode = P32028100457
```

## 1. 接口地址

库存同步接口：

```http
POST http://61.177.139.195:21456//public/syncstock
```

说明：配置文件中的 `host` 末尾带 `/`，插件内部接口路径为 `/public/syncstock`，所以实际拼接后会出现双斜杠：

```text
//public/syncstock
```

一般服务端会正常识别该路径。

## 2. 请求方式

```http
POST
Content-Type: application/json;charset=utf-8
```

## 3. 请求头

```http
appkey: neusoft
nonce: 随机字符串
timestamp: 当前时间戳
sign: MD5(appsecret + nonce + timestamp)
```

签名规则：

```text
sign = MD5(appsecret + nonce + timestamp)
```

其中：

```text
appsecret = f7377865580a02f9a89533b31e4ca7b7
```

## 4. 请求体

请求体为 JSON 数组，一次可以同步一条或多条库存记录。

```json
[
  {
    "synctype": 2,
    "freightbillno": "库存同步",
    "orgcode": "P32028100457",
    "oldproductno": "10801012",
    "oldproductname": "盐酸二甲双胍缓释片（力高欣）",
    "productqty": 3.0,
    "lotno": "1732510114",
    "stockstatus": 1,
    "memo": "",
    "createdate": 1778832064000,
    "purprice": 0.0
  }
]
```

## 5. 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `synctype` | int | 同步类型。`1` 表示普通库存同步，`2` 表示差异/变动同步 |
| `freightbillno` | string | 单据来源，如 `库存同步`、`损溢`、`采购退货`、`销毁` |
| `orgcode` | string | 药店机构编码，当前为 `P32028100457` |
| `oldproductno` | string | GSP 本地药品编码 |
| `oldproductname` | string | GSP 本地药品名称 |
| `productqty` | decimal | 同步数量。差异同步时传差额 |
| `lotno` | string | 批号 |
| `stockstatus` | int | 库存状态，当前插件默认传 `1` |
| `memo` | string | 备注，通常为空字符串 |
| `createdate` | long | 时间戳 |
| `purprice` | decimal | 采购价/成本价 |

## 6. 同步类型说明

### 6.1 普通库存同步

普通库存同步时：

```text
synctype = 1
productqty = GSP 本地库存数量
```

这种方式表示将本地库存数量同步到云药店。

### 6.2 差异/变动同步

差异同步时：

```text
synctype = 2
productqty = 库存差额或本次变动数量
```

例如采购入库、采购退货、损溢、销毁等库存变动，通常传本次变动数量。

## 7. 单条“同步库存”按钮逻辑

在“江阴云药店对接 > 库存同步”页面中，点击列表里的蓝色“同步库存”后，程序会弹出输入框：

```text
请输入云药店中批号为[...]的[...]库存数量：
```

注意：这里要求输入的是“云药店当前库存数量”，不是“本次要同步的数量”。

程序计算逻辑：

```text
上传差额 productqty = GSP 本地库存数量 - 输入的云药店库存数量
synctype = 2
```

示例：

```text
GSP 本地库存 = 10
输入云药店库存 = 7

productqty = 10 - 7 = 3
```

最终上传：

```json
{
  "synctype": 2,
  "productqty": 3.0
}
```

含义：让云药店库存增加 `3`，使云药店库存与 GSP 本地库存一致。

## 8. 输入错误导致失败的原因

因为该输入框要求输入的是“云药店当前库存数量”，程序会根据输入值计算差额。

如果输入错误，上传给云药店的 `productqty` 就会错误。

示例 1：输入值大于本地库存。

```text
GSP 本地库存 = 10
误输入云药店库存 = 12

productqty = 10 - 12 = -2
```

这会向云药店上传扣减库存 `-2`。如果云药店实际库存不足、批号库存不匹配，或扣减后库存不合法，接口可能返回失败。

示例 2：输入值小于云药店实际库存。

```text
GSP 本地库存 = 10
云药店实际库存 = 7
误输入云药店库存 = 0

productqty = 10 - 0 = 10
```

这会让云药店尝试增加库存 `10`，可能导致库存数量错误，或被云端校验拦截。

客户端本地只做以下简单校验：

```text
1. 输入内容必须是数字
2. 拆零库存不支持同步
3. 商品必须已对照云药店目录
4. 输入值等于本地库存时提示“库存数量一致，无需同步”
```

最终是否成功，以云药店接口返回为准。

## 9. 成功响应

成功时接口返回：

```json
{
  "code": 200,
  "message": "处理成功",
  "data": null
}
```

客户端判断逻辑：

```text
code == 200 表示成功
其他 code 表示失败
```

失败时，客户端会显示接口返回的 `message`。

## 11. 查询/检查云药店库存接口

插件中定义了查询/检查云药店库存接口：

```http
POST http://61.177.139.195:21456//public/checkstock
```

插件内路径：

```text
/public/checkstock
```

请求头与库存同步接口一致：

```http
Content-Type: application/json;charset=utf-8
appkey: neusoft
nonce: 随机字符串
timestamp: 当前时间戳
sign: MD5(appsecret + nonce + timestamp)
```

请求体为 JSON 对象：

```json
{
  "orgcode": "P32028100457",
  "oldproductno": "10801012"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `orgcode` | string | 药店机构编码，当前为 `P32028100457` |
| `oldproductno` | string | GSP 本地药品编码 |

插件代码中的返回数据结构为库存列表，字段包括：

| 字段 | 类型 | 说明 |
|---|---|---|
| `oldproductno` | string | GSP 本地药品编码 |
| `productname` | string | 药品名称 |
| `productqty` | double | 云药店库存数量 |
| `lotno` | string | 批号 |

预期成功响应示例：

```json
{
  "code": 200,
  "message": "处理成功",
  "data": [
    {
      "oldproductno": "10801012",
      "productname": "盐酸二甲双胍缓释片（力高欣）",
      "productqty": 7.0,
      "lotno": "1732510114"
    }
  ]
}
```

curl 示例：

```bash
curl -X POST "http://61.177.139.195:21456//public/checkstock" \
  -H "Content-Type: application/json;charset=utf-8" \
  -H "appkey: neusoft" \
  -H "nonce: 12345678" \
  -H "timestamp: 1778832064000" \
  -H "sign: 计算后的MD5签名" \
  -d '{"orgcode":"P32028100457","oldproductno":"10801012"}'
```

代码位置：

```text
GSP.Plugin.JYSI.Service.CheckStockService.CheckStock(string oldProductNo)
```
